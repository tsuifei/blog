CSS 的 OOCSS 設計模式

Posted on 2020-10-25 In CSS

OOCSS (Object Oriented CSS) 最早是由Nicole Sullivan提出的概念，中文直譯是「物件導向的 CSS」但比較接近 「CSS 物件」也就是「可複用」的模式。

OOCSS 有兩個大原則，「結構與外觀(樣式)分離」、「容器和內容物分開」的原則。比較口語化的解釋會直解釋把 OOCSS 以「容器與內容分離」的設計模式來寫 CSS。

結構與外觀分離

定義一個元素的時候，將本身的結構和對元素外觀的樣式分開，增強 CSS 的復用性

容器和內容物分開

最好不要直接使用標籤名定義樣式，減少對 html 的依賴

使用 OOCSS 設計模式來寫的 CSS 框架最有名的算是 Bootstrap，透過「容器與內容分離」樣式的概念達到增加複用與靈活運用的效果與目的。另外一個知名的 CSS 框架 Tailwindcss 也是使用 OOCSS 的設計模式來撰寫樣式。

要開始使用 OOCSS 設計模式來寫的 CSS 樣式時，需要把重複的樣式抽取出來，以及注意命名方法。在早期我們會用語意的方式來為樣式命名，例如.last-articles-box或是.comment-title使用 OOCSS 之後命名會較偏向是例如.links-box或是.tiny-title。

以命名方式來說 常以元素的位置來寫，例如.main .aside {} 表示針對.main元素裡的.aside來撰寫樣式。但是這樣的寫法，會只「針對」這個放在.main裡的.aside，但是.aside如果在其他地方也出現呢？是的，那就再寫一遍，但是通常這樣的區塊風格大多統一或相似，也就是會有許多的樣式會重複寫。

把各種元素拆分成「容器」與「內容」
內容元素表示放在哪裡都可以不被影響樣式

我們可以想像成容器元素就像房子，內容像是房子裡面的家具，放哪裡都可以。
讓各個樣式和其他的元素沒有依賴性，

格線系統，控制欄寬 也是容器類的樣式
